计算机组成原理

计算机组成原理

计算机的硬件结构

  1. 早期的冯诺依曼结构[1]

    提出存储程序的概念：指令以代码的形式事先输入到计算机的主存储器中，然后按其在存储器中的首地址执行程序的第一条指令，以后就按该程序的规定顺序执行其他指令，直至程序执行结束。即按地址访问并顺序执行指令。计算机按此原理应该有5大功能： 数据传输，数据存储，数据处理，操作控制，操作判断。

    根据存储程序控制的概念，由运算器、存储器、控制器、输入设备和输出设备五大基本部件组成，以**运算器[2]**为中心，指令由操作码和地址码组成，指令和数据以同等地位存于存储器，可按地址寻访的计算机称为冯诺依曼计算机。

  2. 现代计算机结构体系

    数据传输无需经过运算器，可以在控制器的控制下，通过输入设备将程序存储在存储器中，再由控制器控制运算器和存储器依次执行每一个计算步骤（指令），最后，控制器控制输出器以各种方式从存储器输出计算结果

    特点：以存储器为中心，将运算器和控制器集成在同一个芯片上，它们在计算机中直接完成信息处理任务，合称中央处理机CPU

  3. 五大功能部件的组成

    ①运算器：

    运算器由ACC(累加器)，MQ(乘商寄存器)，ALU(算术逻辑单元)，x(通用寄存器)组成

    其中ACC用于存储 被加数、被减数、乘数、被除数等

    MQ用于存放乘数和商等

    X是通用寄存器，能存放加数、减数、被乘数和除数等

    以上三个是寄存器，用来存放数据，ALU作为它们的老大，通过内部复杂的电路实现算术运算和逻辑运算

    ②控制器：

    控制器由CU (控制单元)，IR (指令寄存器)， PC(程序计数器)组成

    CU 控制单元，是控制器的核心部件；可以分析指令，给出控制信号，内部集成有复杂的电路

    IR 指令寄存器，存放当前的指令

    PC 程序计数器，存放下一条指令的地址，能自动＋1

    完成一条指令的步骤

    1. 取指令: PC 根据PC的地址取指令 放到IR
    2. 分析指令: IR CU将分析IR中的指令
    3. 执行指令: CU 分析完之后再控制其他的部件配合完成指令

    IR、MAR、MDR是CPU内部工作寄存器，对用户完全透明

    ③存储器：

    主存储器(CPU能直接访问)和辅助存储器(外存储器，容量大，但是需要将信息读入主存中才能被CPU访问)；

    主存储器是按照存储单元的地址寻址的，也称为按地址存取。

    主存储器由 MAR、MDR、存储体和时序控制逻辑构成

    MAR 有k位，不一定要完全使用这k位 ，比如说有MAR 有32位，MDR有8位（一个字 取 1B），2^32*8 = 4GB ，也有可能存储器只有1GB的空间。例题: 8位的计算机用16位表示地址，则有2^16 个地址空间，不一定全用

    ④输入设备 && 输出设备

    输入输出设备统称为I/O设备，CPU + 主存储器 + I/O设备构成了线代计算机的三大部。CPU和主存构成计算机的主体，称为主机或处理机。相应地把I/O设备叫做外围设备或外部设备，简称外设。

  4. 一条指令的具体执行流程以y=a*b+c为例：

    首先 PC 先指向第一条执行的指令的位置（0），再将这个位置给MAR 由MAR去存储体中寻找对应地址的数据，如何将该数据放到MDR中。再传到CPU中的IR，IR中的数据拆分成 操作码和地址码，CU拿到操作码，分析是什么动作–读数– 将地址码放到MAR，MAR取地址中的数据给MDR，MDR再将数据放到ACC累加器中

    总结：指令的执行过程👆 PC –> （MAR + MDR …）–> IR –> CU –> 具体的执行步骤

  5. CPU区分指令和数据的依据

    指令和数据都是用二进制数码形式分区域存放在存储器中，为了能区分从存储器中读出的二进制数码是指令还是数据，可以用下列方法判断：

    凡由程序计数器提供的存储单元地址中取出的二进制数码是指令，应送到指令寄存器中；

    凡由指令中地址码部分提供的存储单元地址中取出来的二进制数码是操作数，一般应送到运算器中。

计算机的软件结构

系统软件：管理整个计算机系统：操作系统、数据库管理系统、标准程序库（库函数）、网络软件（实现TCP/IP协议）、语言处理程序（编译程序、汇编程序）、服务程序（DeBug 程序）

计算机的多级层次结构

  1. 计算机的五层结构

    在上一节中：在对 y=a*b+c 这条指令进行执行时，先是 将它转化成 机器代码 –> 机器代码执行时有分成了 1. 取数 2. 乘 3. 加 等等操作，在这些小操作中，落实到具体的机器上又分成：1.从PC中取地址->MAR 2.M(MAR)->MDR 3.MDR->IR 等步骤，我们将 00000100000000101 称为指令，将它拆分成多个小步骤称为微指令，硬件直接执行微指令。

    ① 第1级为微程序级。它是由硬件直接实现的，执行微程序指令，是计算机系统最底层的硬件系统。

    ② 第2级为机器指令系统级。又称传统机器级，执行二进制指令。 通常将第1、2级统称为传统机器级。

    ③ 第3级是操作系统级。计算机系统中硬件和硬件资源由该级管理和统一调度，它向上提供广义指令(系统调用)。

    ④第4级是汇编语言级。

    ⑤第5级为高级语言级。

    [注 : 编译汇编和解释程序统称为翻译程序]

    三种级别语言的转化：有的高级语言要先编译成 汇编语言 再进一步 由汇编器编译成机器代码，也有能直接通过编译器编译成机器语言的高级语言。除了编译程序之外，还有解释程序，每读取一句就立即执行

计算机性能指标

  1. 存储器的性能指标 (见 五大功能部件 — 存储器)

    总容量 = 存储单元个数 * 存储字长 bit （* 存储字长/8 Byte）

    存储单元的个数最多有 = $$2^k$$ 个 (MAR 的位数 为 k) 存储字长 = MDR的长度 (字长)

  2. CPU的主频

    CPU内数字脉冲信号振荡的频率 __|￣|_|￣|__|￣__|￣|_ 每一个周期称作CPU的时钟周期

    $CPU主频 = 1/CPU时钟周期$

  3. CPI – 执行一条指令所需要的时钟周期数。不同的指令，CPI有可能不同，例如取数指令要9个周期（一个周期对应一个微指令），乘法指令要11个周期。所以，我们需要一个公平的指标来衡量：执行一条指令的耗时 = CPI * CPU的时钟周期 = CPI / CPU 主频。eg: 假设有100条指令 ，CPI是 3，CPU的主频是1000HZ，问执行完这100条指令要花费多久？ 100 * 3 / 1000= 0.3 s

  4. IPS ： 1秒内能执行多少条指令 = 主频/平均CPI 。

  5. FLOPS： 每秒执行多少次浮点运算。

  6. 在计算机的性能时，$K，M，G，T = 10^3 , 10^6, 10^9, 10^{12} $

  7. 数据通路带宽： 数据总线一次能传输位数

计算机中数值的表示格式

计算机中数值分为 定点数和浮点数，没有用来表示小数点的元器件，小数点一般隐含在数中

无符号数和有符号数：

n位无符号数的表示范围 全0 ~ 全1 ：00000000 ~ 11111111 -> 0~255 = ( $$2^8 -1$$)
n+1位有符号数的表示范围：有符号整数：最高位为符号位 （0正 1负）剩余n位范围为 $$0 到 2^{n}$$ ，由此可见 有符号整数（原码）： 【1(负数) $$2^{n}$$ 到 负0; 正0 到 正 $$2^{n}$$】
**n+1位有符号小数(纯小数)**：一样同上 数码部分(n位的话)全0 ： 0 ，全1： $$2^{-1}+2^{-2}+2{-3}+…+2^{-n} = 1-2^{-n}$$ 范围是同样有 正0 和负0之分 【1(负数) $$1-2^{-n}$$ 到 负0; 正0 到 正 $$1-2^{-n}$$】

定点数： 分为定点小数和定点整数

定点小数：

小数点隐含，固定在最高数据位的左边，整数位则用于表示符号位0正 1负，用于表示纯小数

例如: + 0.001 –> 0.001 | -0.001 –> 1.001

定点整数：

小 数点位置隐含，固定在最低位之后，最高位为符号位 0正 1负，用于表示纯整数

浮点数：既有整数部分也有小数部分，可以通过改变阶码来改变小数点的位置
例如：计算机中的 $$(1101.011) = (0.1101011)* 2^{100} = 13.375$$ 阶码就是科学计数法中10的x次方中的x，阶符就是阶码的正负，数符表示数的正负，阶码部分表示浮点数的表示范围，尾数部分表示浮点数的精度

原码、反码、补码和移码

对于有符号数我们采用下面几种表示方法：

原码：最高位表示符号，其余各位表示数的绝对值(正常表示)

反码：对于（符号位为0）正数，反码是它本身；对于（符号位为1）负数，反码是数值位取反 【从定义可以看出，反码依旧没有解决+0 和 -0 的表示问题0,0000 = 0；1,0000=1,1111,其实反码是原码转化成补码的中间状态，没什么用，重点还是看下面的补码！】

补码：正数（符号位为0）的补码是它本身，负数（符号位为1）的补码为反码末位+1 （小数要在最后一位加1）。

关于补码在+0 和 -0 的处理方法上的分析：补码的正0 为它本身 0,0000000，－0 的补码是：[1,0000000]_原 –> [1,1111111]_反 –> 末位加1 [0,00000000] (最高位符号位的进位被舍弃)。这样下来原码的1,0000 和 0,0000 的补码是一样的，那么我们就把后者作为0正数的补码是它本身，前者规定为 2^k k为数值位的位数，例如：$$1,0000000 = -2^{7}$$，这样一来补码的范围就从原来正常的范围将其中一个重复的0给到负数最小一位如下图所示：

同理对于小数的 1.0000000 我们规定它为-1；0.0000000 表示为0

补码转换成原码的方法：负数：尾数取反再在末位+1 或者 负数补码最右边的1及其右边同原码保持一致，最右边的1 左边同反码保持一致(不包括符号位)

例如：10011000是补码，它的原码为：数值位取反 +1 ，1,1100111 +1 = 1, 1101000；或者 1,0011000 最右边的 1 及其右边和原码保持一致：1000 它的左边和反码保持一致，取反得到 0,110；组合起来就是：1,1101000

已知x的补码，求-x的补码：符号位和数值位全部取反，然后末位+1

移码： 在补码的基础上将符号位取反 且移码只能作用于整数。移码的表示范围和补码一样 $$-2^{n} 到 2^{n}-1$$ 且移码和补码都只有一个0

为什么要有移码！？？？

通过列出所有的移码，我们不难发现：当我们所表示的数在不断增大时，移码的值也在不断增大，因此，在对二进制整数比较大小时，从左往右，只要谁先出现了1，谁就更大，这样的特性很方便的让计算机对整数进行比较。（下文的 浮点数运算中阶码就是用移码表示的）

为什么要有这些奇怪的编码?

相反数相加的问题

如果只有原码，会产生：溢出、+ - 0 、等问题

对于有符号的二进制数相加减运算时，符号位 1 + 0 有时不产生进位，【原码相反数之和不得0，正数加正数有可能得到负数，即上溢出，负数加负数也可能得到正数，下溢】就意味着不会得到正确的答案,要实现这个功能，我们要设置加法器或者减法器，加法器设计成本低，但是减法器复杂、成本高。所以可以尝试用加法代替减法。

这里提出模运算等价的概念：例如时钟有12个点，我们将所有的时间对12取余 – x mod 12，根据得到的余数不同将数分成12类（余数分别为 0 ~ 11），这样就意味着 12 -3 和12 +9 等价【9%12 = 0*12 + 9；-3%12 = -1*12+9 】。这里引出补数的概念：补数 即两个数的绝对值之和 = 模 （abs(-3) + abs(9) = 12）

补数的性质： 在时钟里 一共有12个值，10 -3 = 10 + 9；在9位有符号整数中， 任何运算结果在 mod 2^8 后只保留最低8位，如果要进行减法运算 x-a = x+(-a)，可以用模 - a的绝对值 = a的补数，这里给出：a的补数就是a的补码 [这里是要求 （-a）的补数，然后与x相加就能得到原码的正确形式的加法]

补码：让减法操作转变位加法操作，节省硬件成本

定点数的运算: 移位、加减和乘除

算术移位：

左移和右移

数的高位相对于小数点进行位置移动，靠近小数点就是右移，远离小数点就是左移。右移可以看作是除以2并且舍弃低位（数位只有这么多，丢失精度），左移看作是乘2并且舍弃高位，若高位不为0，就会出现严重误差。0100 右移是 0010。0100 左移是1000。这是在原码的形式下移位，正数反码和原码一样，补0，负数就相反，左移和右移都是补1（1代表0）。对于补码来说，正数和反码 原码一致，移位时补0，负数就是最右边的1 和它右边同原码一致，补0（这里是乘法），最右边的1的左边和反码一致，（除法）补1。

逻辑移位：

相当于无符号的算术移位，有符号数符号也会跟着左移或右移。左移就是低位补0，舍弃高位；右移就是高位补0，舍弃低位。例如表示RGB的值，使用3B的大小存储3个色彩值，先让3B的低位存储R的值，然后左移16位，就把R的值存储到了高8位；再加上G的值，左移8位；然后是加上B的值，就完成了从左到右存储RGB的值。

循环移位：

数据左移的时候，左边多出移位，原本被舍弃，但是现在不在低位补0，将多出的那个高位放到低位。实现循环补位。如果是含有进位位的计算机，记录低位的运算有没有产生进位，如 果产生就会把1放到进位位，原来进位位中的值复制到最低位。

**加减: **

原码的有符号数由加法器直接相加，但是有可能出错—溢出。由于减法操作如果使用硬件实现非常复杂，因此，计算机一般使用补码来进行加法运算，数值位与符号位同时参加运算。再取原码就得到了正确答案 ↑回看上面的加法操作[相反数相加]

判断溢出：

溢出的情况有： 正数＋正数 > n位数所能表示的范围，负数+ 负数 < n位数所能表示的最小值。

解决办法：

  1. 根据符号判断：表达式：A+B = S，如果满足逻辑表达式 A 的符号 & B的符号 & !(S的符号) || !(A 的符号) & !(B的符号) & (S的符号) 就说明溢出。原理是 a. 负数1 和 负数1 ，如果相加溢出，S的符号位就是正的，这就对应前面的表达式，如果成立就说明下溢出。 b. 正数0 和 正数0 如果溢出就是负的，因此S的符号为1 这样就是后面的式子，这个表达式成立就说明上溢出。
  2. 根据符号位的进位判断：符号位的进位为0，最高的数值位向前进位为1 就说明上溢出了（符号位原来是0，进位1，是由最高的数值位进位过来的）。若符号位的进位为1，最高值进位为0，就说明下溢出
  3. 双符号位表示法： 正数00 负数11，如果产生进位：00 + 00 = 01 || 11 + 11 = 10 符号位前面一位表示正确的符号位，后一位表示当前得到的符号位。

符号扩展：

整数： int 扩展为long时，正数的原码、反码补码都是补充0，负数原码扩充的位补0，反码和原码相反，补充1；而补码找最右边第一个1，这个1 左边的数和反码保持一致，这个1（包括）右边的数和原码一致。

小数：正数和负数的原码都是在后面添加0，负数反码是后面补1，补码是最右边第一个1同其右边和原码保持一致，其左边部分和反码一致。

原码乘法：

原理：跟十进制类似，A*B = 28*36 = 28*6+28*30 = A乘 B的每一位然后乘位权。那么在二进制也一样：

0.1101不断乘1011 中的位（1或者0），然后累加，累加之后需要逻辑右移（有可能会进位到符号位，符号位也要跟着右移，算术移位就不用移动符号位），不断把MQ中已经运算过的B的低位舍弃，记住B的符号位不用运算，直接舍弃就行（直接右移ACC中的内容）最后ACC和MQ拼凑起来就是结果。两数的符号做异或运算 0 ⊕ 1 = 1。

大端存储： 地址由低到高，依次存储位权减小的数，和我们现在的阅读习惯一样 875，先看的是大的数800 再是70 最后是5。

小端存储：地址由低到高，依次存的是位权增大的数，先看的是小的数，再看的是大的数 低 5 7 8 高 便于机器存储

浮点数：

浮点数的真值： $N = r^E * M$ ，r是基数 默认为2

浮点数有可能出现： 8位（1B）的存储空间，阶码 0 10，01101 在存储时会超出8位，如果1 被舍弃，则损失精度。因此要在有限的长度内尽可能减小尾数精度的损失，我们对尾数进行规格化。以十进制科学计数法为例子：我们规定尾数部分最高位一定是一位有效位，类比于2进制就是尾数第一位一定是1，这样就减小了前面空着没有意义，后面需要保留精度的数被舍弃的概率

存储系统

层次结构：

按照速度 从快到慢：CPU中的寄存器> Cache>主存>磁盘>磁带 、光盘

cache 到 主存 这级解决了主存速度与CPU速度不匹配的问题，主存和辅存 这一级解决了主存容量不够的问题（虚拟存储器）

按照存储介质分类：

半导体存储器：主存、cache 采用半导体元器件存储数据，速度极快

磁性存储器：软盘、磁带、磁盘（机械硬盘）采用磁性材料制成，速度稍慢

光存储器： DVD CVD 光盘，以光介质存储信息

还可以按照存取方式分类：

随机存取：读写任何一个存储单元所需的时间都相同，与位置无关，例如固态硬盘，内存

顺序存取：磁带（录音带，必须顺序的读取，读取某个存储单元所花费的时间取决于它所在的位置）

直接存取存储器：既可以随机存取也可以顺序存取，机械硬盘：先选择读哪里，磁头移动–>找道–>磁盘旋转，当要读的数据转过来就能读出存储的内容

按内容访问：相联存储器，可以按照内容检索道存储位置进行读写，快表就是一种相相联存储器

前三个都是按地址访问，最后一个按地址访问

根据所存储的信息是否能被更改

读写存储器：可读 也可写（磁盘，内存，cache）

只读存储器（ROM）：例如音乐专辑 CD-ROM，电影的蓝光光碟，以及BIOS也是写在ROM中

根据所存储的信息可保存性分类：

断电后，存储的信息立刻消失–易失性存储器（主存、cache）
断电后，存储的信息依然能保持的存储器–非易失性存储器（磁盘，光盘）
信息读出后，原来的信息被破坏–破坏性读出，例如DRAM 芯片，读出后要重新写入（重新加高低电平，形成电子）
非破坏性读出 – SRAM

存储器的性能指标

注意：

存储速度：存储字长（存储单元的字长）/存储周期
存取周期指一次完整的读写操作所需要的时间，连续两次独立的访问存储器之间最小的时间长度（间隔）因为要算上恢复时间

存储器读写数据的原理

主存中，每个存储元有一个mos管开关和一个电容，多个存储元和并成一组，叫存储字，红线连接一组存储元的mos管，当在红线上加高电平，就能导通mos管，电容中的电荷顺着绿线导出，就能根据是否有电荷判断存储的是0还是1。因此存储器读写的单位是存储字。一般一次读8位16位或32位64位等等。

CPU和存储芯片的连接

  1. 存储芯片读取数据： 存储器通过地址总线与CPU相连，MAR给出一个地址到地址总线上，这个地址经过译码器可以映射到一根字选线，对应一个待读取/写入的存储字，读取的存储字会经过数据总线读出。 例如 MAR中的是00100100 表示第 36根片选线。MAR中有n位表示一共有$$2^n$$个存储单元。读控制线和写控制线 表示当前是读操作还是写操作 ：两根：$$\overline{WE}$$ （低电平） 允许写，$$\overline{OE}$$允许读。一根表示读写：$$\overline{WE}$$ 低电平写，高电平读。片选线：作用是在多块存储芯片构成的存储器中，通过片选线控制读取哪块芯片。

    求存储芯片的容量： n位地址 2^n 个存储单元，每个存储单元对应一个存储字，例如 8k*8位的存储芯片是 8个存储单元，每个存储单元对应的存储字是8位， $$2^{13}*8bit$$

  2. 寻址：现代计算机一般是按字编址【也有按字节寻址，按半字寻址，按双字寻址】，一个存储字就是一个地址，在存储矩阵中一行表示一个字，我们如果按字寻址的话，容量/字长 得到对应的总的地址空间，如果是按半字寻址的话，容量/(0.5*字长）= 地址空间大小。按双字寻址就是 容量/(2*字长)。

    例如： 按字寻址，字长为4B，容量为1KB，1KB/4B = 2^8 个地址空间。按字节寻址：1KB/1B = 2^10 个地址空间，按双字寻址 1KB/8B = 2^7 个地址空间。根据地址空间大小确定地址的长度 例如按字寻址的地址是 可以算术移位得到下一个地址的起始地址：1 –> 100 相当于左移2位 （4/2）。–根据个人理解得出的结论

  3. DRAM用于制作主存，是用电容保存电荷来存储数据，每次读取数据时就导出里面的电荷，当然也要重写回去。当计算机掉电后主存存储的内容就会失去，因为电容中的电荷会泄漏，所以要周期性刷新。SRAM是用于制作Cache的芯片，它是电荷存储方式，也是易失性存储器。但是它是非破坏性读出，不需要重写。【SRAM的缺点是每个存储元制造成本更高，集成度低，功耗高，但是读写非常快；DRAM的缺点是读写速度慢，还要刷新，但是每个存储元制造成本更低，集成度高，功耗低】

  4. 上面的DRAM有一个非常大的弊端，就是MAR位数大的时候，译码器要连接非常多根选通线，n = 10， 2^10 = 1024 根选通线，于是我们把一维的地址转变成2维的，将存储单元排列成矩阵的形式，地址前半部分为行地址，后半部分为列地址，行地址由行地址译码器译码，列地址由列地址译码器译码，这样就能大大缩小译码器的选线数量：8位 = 256根 4+4 = 行16+列16=32根选通线 ，每一个行地址和列地址组合对应一个存储单元【存储字】

  5. DRAM的刷新 大约是2ms一次，我们一般以行为单位，一次刷新一整行，先读出后写入，所用时间是一个读/写周期。

    刷新方式
    分散刷新：每次读写完都刷新一行
    集中刷新：在2ms内最后一部分时间里安排全部刷新
    异步刷新：为保证2ms内全部n行都能刷新，那么2ms/n = x ms，这样我们只需要在x ms 内刷新一行，即可保证2ms内所有的行都被刷新过。
    假设DRAM 是128*128的结构，读写周期是0.5us，在2ms内 采用：
    1, 集中分散刷新：存取周期变成原来的2倍
    2, 集中刷新2ms最后有 128*0.5 = 64 us 用于刷新，剩下的 2000 - 64 = 1936 us 可以用于读/写数据。64us也称为死区，不能读写数据
    3, 异步刷新：2ms / 128 = 15.6 us，每15.6 us内拿出0.5 us用于刷新
